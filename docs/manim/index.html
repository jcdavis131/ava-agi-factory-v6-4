Why Manim for Ava?

Previous Ava dashboards used static SVG flow W1380 H780 COLS 110/230/... with 1:1 input→tower cat box. Now we use Manim — Python code → precise vector graphics, same engine as 3Blue1Brown. Programmatic, time-accurate, LaTeX-ready (though we use Text for speed since texlive optional in VM). Each animation is code: class MTNNFlow(Scene): → Write(), Create(Rectangle), Arrow(..., stroke_width=4), Dot.animate.move_to() showing data flow. OpenGL via moderngl 5.12.0 + moderngl_window + PyOpenGL, Cairo via manimpango.

Maps to 6 local LLMs review: Zaya1 8.4B MoE 760M active compressed conv 8× KV 64GB→8GB, Qwen 3.6 Gated DeltaNet fixed 18MB @170K never grows 3:1 precision checkpoint 25-40 tok/s, DeepSeek V4 Flash 284B 13B active 1M ctx 10% KV disk ds4 C, VibeThinker-3B $7,800 verifiable reasoning packs smaller, DiffusionGemma 26B/4B-active 256 canvas 123 steps bidirectional, Gemma4 E2B/E4B MatFormer + per-layer embeddings 6GB RAM 7-8 tok/s.
